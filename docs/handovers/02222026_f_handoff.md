### Architecture Decisions

- **Table prefix**: All Meridian DB tables use `meridian_` prefix to avoid collisions with existing Waypoint tables (`students`, `campuses`, etc.)
- **Reused `profiles` table**: Meridian's original `staff` table replaced with FK references to existing `profiles`
- **RLS pattern**: Reused existing `user_district_id()` and `is_waypoint_admin()` PostgreSQL functions (no new functions needed)
- **Views include `district_id`**: Both computed views (`meridian_student_folder_readiness`, `meridian_compliance_deadlines`) include `district_id` so hooks can filter client-side
- **Theme**: Meridian uses purple accent — `bg-purple-600`, `text-purple-600`, sidebar active state `bg-purple-600`
- **No react-hot-toast**: Action buttons use `alert()` as placeholders (matching pattern from other pages)
- **`activeProductCount`**: Sidebar now counts all three products to decide whether to show section headers
