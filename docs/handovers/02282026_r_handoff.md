### 1. clearpathedgroup.com 500 Error — Root Cause + Fix

**Root cause:** The `node deploy-clearpath.mjs` Direct Upload API script was creating deployments that returned HTTP 500. Cloudflare's Direct Upload API creates deployments that never properly initialize the CDN serving layer when the project was created via API (not the dashboard). The script also deleted and recreated the project during debugging, which removed the custom domain bindings.

**Fix:** Committed and pushed `clearpath-site/index.html` to git → triggers `.github/workflows/deploy-clearpath-site.yml` → uses `cloudflare/pages-action@v1` (wrangler-based) → properly activates deployments. Same method as the working `waypoint` project.

**CRITICAL:** Never use `node deploy-clearpath.mjs` (Direct Upload API) again. All clearpath-site deployments must go through GitHub Actions (push to main or `workflow_dispatch`).
