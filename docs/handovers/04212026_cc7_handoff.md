## Lessons worth keeping

- **Browser IndexedDB is origin-scoped.** Any product that exists as both a hosted web app and a downloadable standalone HTML will orphan user data on the transition unless the UX explicitly bridges it (Export → Import). Same risk exists conceptually for Beacon if we ever ship a "download your Beacon for air-gapped use" mode. If we do, same bridge note applies.
- **Multi-chunk Vite bundles are only safe under their originating Cloudflare Pages deployment.** Toolkit's `/downloads/...` file is safe because it's an inline single-file. Beacon's `/downloads/...` file was NOT safe because it was a multi-chunk bundle dropped into a different origin's `/downloads/` folder. The detection check: does the HTML have `<script src="/assets/...">` or `<script type="module" src=>` tags? If yes → broken when served outside its own origin. If no (only inline `<script>` with base64) → safe.
