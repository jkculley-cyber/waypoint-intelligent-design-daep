- **§37.0041 doesn't exist** — MDR procedures are under **§37.004(b)** + IDEA 34 CFR §300.530. Fixed in `clearpath-site/whitepaper.html`.
- **§37.005 mislabeled as "Removal by teacher"** (which is §37.002) and as the offense definition for Insubordination (which is §37.001). Fixed in `src/lib/originsScenarios.js` and `docs/origins-content/parent-content-library.md`.
- **§37.007 cited as "Discretionary DAEP"** — §37.007 is mandatory expulsion. Discretionary DAEP is §37.006(a). Fixed in `whitepaper.html`.
- **SB 179 conflation** in `counselor-caseload-tracker.html`: SB 179 of 2017 is David's Law (cyberbullying, §37.0832); SB 179 of 2019 is the counselor 80/20 bill (§33.006). Disambiguated.
- **§37.013 used as catch-all parent notification** in 6 build-tpt lines — §37.013 is the noncustodial parent / conviction notice statute, not general parent contact. Replaced with **§37.009(a) + local Board policy (typically TASB FO/FOC)** framing.
- **HARASS-02 sexual harassment** cited as `TEC 37.006(b)` (which is employee retaliation, not sexual harassment). Replaced with `TEC 37.001 / 37.083; 34 CFR Part 106 (Title IX)`.
- **TAC §97.1071** wording softened in `research.html` (date unverified).
- **investigator-toolkit/intake.js**: §37.0052 (doesn't exist) → §37.0832 (David's Law); drugs subsection §37.006(a)(1) (was actually terroristic threat) → §37.006(a)(2)(C)–(D); threats §37.007(a)(3) (felony drugs) → §37.006(a)(1).

## Lessons worth keeping

- **jsPDF + WinAnsi encoding is flaky for any non-ASCII char** (em dash, middle dot, multiplication, arrows, checkmarks). Replace with ASCII at the entry point before passing to `doc.text()` or autoTable bodies. Inline color-coding via `didParseCell` is a clean alternative for status columns.
- **PWA service worker caching is a recurring footgun for users testing new features.** Two times this session the user reported "X isn't working" after a deploy because their browser was serving the old cached bundle. Hard refresh + DevTools → Application → Unregister SW solves it. Worth adding to a customer-facing "having issues?" doc.
- **Hooks-after-early-return crash** — added a useEffect for TinyURL fetch BELOW the `if (!open || !counselor) return null` guard, broke the modal entirely. React's rule of hooks: every hook runs in the same order on every render. Always lift hooks above any conditional return; compute derived values with `?.` chaining if needed.
- **`hold off on TX-NDPA`** is a useful pattern from the user — explicit gating on the touchiest legal/business decisions while shipping the rest of the work. Need to track these explicit holds carefully so they don't go missing.
- **Director-of-counseling sales tool requires drillable detail.** The first version of the District Preview KPIs were just static numbers. User immediately said "should be clickable." Sales tools demoed live in a board meeting need depth on demand — every number should answer "where does that come from?"
- **TSCA CREST aligns to the 5 categories of the Texas Model.** Once mapped, much of what Beacon already collects (Time Tracker → Cat-2, Lessons → Cat-5, Sessions/Referrals → Cat-4) directly feeds the portfolio. Auto-populate is the obvious Phase 2.
