## Lessons worth keeping

- **Hide aspirational features in code, not just intent.** The Cloud-mode toggle was flagged as "Hidden-in-spirit" 6 days before this session; the inventory note was the only thing keeping a curious counselor from a broken state. `{false && (...)}` with a restore-condition comment is the minimum acceptable surface.
- **The CC8 modal-reset rule (DECISIONS.md 2026-04-23) applies to NEW modals immediately.** ConfirmDestructive was built with `useEffect([open])` from the first commit — no rework needed.
- **`localStorage['beacon_last_backup']` is a shared signal across multiple consumers.** The new modal threads the same write so the AppShell banner, Scorecard, and OnboardingChecklist all stay coherent. Worth checking before adding another backup path.
- **Defense-in-depth ≠ replacement.** The backup-first prompt sits ON TOP of the CC8 tag-and-filter fix, not instead of it. If a future seeder forgets to tag rows, the backup checkbox is the parachute.
