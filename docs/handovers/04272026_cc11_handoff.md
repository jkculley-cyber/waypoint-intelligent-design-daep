1. **Verify CC10 + complete CREST Phase 2.** User wanted CC10 deploys validated and the Phase 2 follow-ups (auto-populate, portfolio PDF, file attachments, District Preview rollup) shipped from the CC10 deferred list.

2. **Strategic question: what would make Beacon something a counselor can't live without?** User asked for an honest, transformative answer — not "what would please me." The answer was three killer features: **voice capture**, **crisis workflow**, and **daily morning brief**. The user then directed building four (#1 PWA alerts + ICS, #2 Crisis, #3 Parent Comms, #4 Morning Brief) plus the local-mode session-note template expansion (#5). Voice capture (the biggest one) was scoped but deferred — needs a 2-3 week build for whisper-wasm integration and is best done after the lower-risk features are stable.

Voice + FERPA was the central security question of the session. Verdict: only **whisper-wasm running locally on-device** is FERPA-safe for the $79 individual buyer. Web Speech API and OpenAI Whisper API both transmit audio to vendors without a DPA, which is quietly violating for individual buyers. Audio is captured by `MediaRecorder`, transcribed in a Web Worker, and discarded by default. The model file (~80 MB) is the only network call, downloaded once and cached.

**Schema** — IndexedDB v4 → v5 with 4 new stores in one migration: `crisis_events`, `parent_contacts`, `follow_ups`, `session_note_templates`.

**#1 Native PWA alerts + ICS export** (`src/lib/notifications.js`, `src/lib/calendarExport.js`, `public/sw.js`, SettingsPage RemindersPanel)
- Service worker handles notification clicks + posted-message firing.
- In-app poll (60s + on-focus) fires browser notifications for upcoming sessions and pending follow-ups when PWA is open.
- ICS download with anonymized titles by default; opt-in to include student names with FERPA warning ("may sit in your calendar vendor's cloud").
- All on-device; no push server.

**#5 Local-mode session-note templates** (`src/lib/sessionNoteTemplates.js`, `components/SessionNoteTemplateModal.jsx`)
- 10 SOAP templates: regulation, grief, academic, family, bullying, friendship, trauma, mood, goal-setting, closure.
- "Use template" button in Sessions QuickLogModal → picker → 4-6 prompted fields per template → live SOAP preview → stitches into the notes field.
- No AI; pure template stitching with sentence-grammar polish.

**#4 Daily Morning Brief** (`src/lib/morningBrief.js`, `components/MorningBriefCard.jsx`)
- Dismissible card at top of Dashboard, shows once per day (state stored in `settings.morning_brief_dismissed`).
- Aggregates: today's sessions + recurring blocks + overdue follow-ups + this-week SB 179 hours-to-target + CREST deadline + open referrals.
- Distinct from existing "My Day" priority view (which stays granular Tier 2/3-focused). Brief is the executive zoomed-out view.

**#3 Parent Communication Hub** (`src/lib/parentContacts.js`, CommunicationsPage)
- Outcome enum: answered / voicemail-left / no-answer / parent-acknowledged / meeting-scheduled / no-response / certified-mail-sent / returned-undeliverable.
- USPS tracking number field auto-shown when contact_type=Certified mail or outcome=certified-mail-sent.
- Auto-creates a 3-business-day follow-up (`follow_ups` store) when outcome is unanswered.
- Escalation banner on the form when the selected student has 2+ unanswered attempts in the last 14 days ("recommended" at 2, "urgent" at 3+ with certified-mail nudge).
- Due-Process Documentation PDF per student listing every attempt with timestamps + counselor signature block. Defensible record for due-process / SPED / OCR proceedings.
- Outcome + tracking-number badges shown on contact history rows.

**#2 Crisis Quick-Launch + CPS workflow** (`src/lib/crisisWorkflow.js`, `src/lib/crisisExport.js`, `components/CrisisModal.jsx`, `components/CrisisLaunchButton.jsx`)
- Floating red `🚨 Crisis Now` button on every authed page (z-index 250, bottom-right).
- 4-stage modal: picker (student + trigger) → workflow (per-trigger structured questions) → review (with auto-drafted parent + admin notifications) → done (PDF generation).
- Four trigger types each with their own 9-15 step workflow:
  - **Suicide / self-harm** — 14 questions inspired by Columbia Protocol (original wording, not literal C-SSRS), with risk-level determination + 988 callout.
  - **Threat to others** — SB 11 / 19 TAC §103 classification (transient vs. substantive serious / very serious / imminent) + protective actions + threat-team consultation.
  - **Abuse / neglect** — mandated-reporter flow with DFPS hotline 1-800-252-5400 prominent + intake worker / case-number capture + parent-notification warning per CPS guidance ("not contacted — would compromise investigation" as default option).
  - **Acute distress** — de-escalation log with mandatory safety check (escalates to suicide screen if student endorses).
- On save: writes to `crisis_events`, auto-schedules 24h/72h/1-week follow-ups in `follow_ups` (each becomes a future Morning Brief signal + browser notification), drafts parent + admin notifications (counselor copies/pastes/emails), generates timestamped defensible PDF for the confidential file.

## Decisions logged (2026-04-27)

Five new entries to DECISIONS.md:

1. **Voice capture for individual buyers MUST be local-only Whisper-WASM.** Web Speech API streams audio to Google/Apple/Microsoft cloud; OpenAI Whisper API streams to OpenAI. Both are quietly FERPA-violating for individual buyers without a DPA. Local Whisper runs in a Web Worker with `MediaRecorder` audio held in JS memory only and discarded after transcription (default). Model file (~80 MB) is a one-time network call that contains no student data.
2. **Crisis screen workflows are inspired by Columbia / SB 11 but use original wording.** Beacon does not bundle the literal C-SSRS form (which has its own license + use rules). Districts that need literal C-SSRS use that form directly. Beacon's structured screen captures the same risk dimensions in a defensible documented format.
3. **Native browser notifications via service worker postMessage; no push server.** Counselor enables alerts in Settings → permission prompt → in-app poll fires while Beacon is open. ICS download is the secondary persistence channel for desktop calendar apps. No Google OAuth, no Web Push, no third-party.
4. **Beacon's path from "good binder" to "indispensable" is three transformative features:** (a) voice capture (Whisper-WASM, deferred), (b) crisis workflow with auto-documentation (shipped this session), (c) daily morning brief (shipped this session). These three are the conviction-buy features that move per-counselor pricing from $79 curiosity to $200 must-have, and unlock district adoption. All other backlog items (504 module, SEL screener, translation, "what's next" tile) are widening the moat but not transformative on their own.
5. **Crisis follow-up auto-scheduling is non-negotiable.** Suicide and threat workflows auto-create 24h + 72h + 1-week reminders. Abuse creates 24h + 1-week. Acute distress creates 24h + 72h. These reach the counselor through the Morning Brief card AND browser notifications (when enabled). Missed follow-ups are the single largest counselor lawsuit risk; making the schedule mandatory is the lawsuit-insurance value proposition.

## Lessons worth keeping

- **Schema bumps are best batched.** v4 → v5 added 4 stores in one migration covering 4 different features. Trying to do them in 4 separate migrations would have made the rollout painful for any counselor on her phone.
- **Modal-reset rule (CC9) is now a sweep target.** Crisis Modal needed `eslint-disable react-hooks/set-state-in-effect` for the modal-reset effect. Acceptable per CC9 decision but lint flags it. Worth a sweep to convert these to `key={open}` remount-style or to a custom `useResetOnOpen` hook.
- **Counselor product = lawsuit insurance + time-back.** The crisis workflow is THE feature counselors will pay $200 for. It's not "useful" — it's "the thing that keeps me from losing my license." Frame all marketing on this rather than on organizational features.
- **Each crisis workflow takes ~5 minutes to walk through.** That's the right ceiling. If the counselor is in actual crisis-response mode she doesn't have 15 minutes; she also doesn't trust 30-second tools to be defensible. The 5-minute structured screen with auto-followups + PDF is the magic number.
- **The Morning Brief is the open-the-app-every-day hook.** Without something that pulls the counselor in at 8 AM, Beacon becomes a write-only tool she opens when she remembers. The Brief is the "open me first thing" anchor — that's how Apex's morning brief works for principals and how Beacon's needs to work for counselors. Worth A/B testing the Brief copy + signal-selection over the next quarter.
- **`docs/legal/TX-NDPA-vendor-prep.md` and `clearpath-site/dpa.html` are STILL uncommitted from CC10.** Two sessions in a row now. User explicitly asked to hold; should not be auto-committed without confirmation.
