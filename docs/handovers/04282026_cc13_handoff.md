### Attestation chain — actually works end-to-end now (`71f2a06`)
- The CC12 round-3 implementation used `Prefer: return=representation` which makes PostgREST do INSERT-then-SELECT. Audit-log table grants anon INSERT but not SELECT (intentional — anon shouldn't enumerate hashes), so SELECT failed with `42501 row-level security policy` and the whole call returned null. Every PDF was printing "Attestation: offline at generation time (skipped)."
- Fixed: generate receipt UUID client-side, send `Prefer: return=minimal`. Server's `DEFAULT now()` sets the authoritative timestamp; verify URL surfaces it via service-role read. Verified HTTP 201 against live ops Supabase.
- Skipped sending `generated_at` from client entirely so a client with bad clock can't fail the new CHECK constraint AND can't spoof the server timestamp.

## Decisions logged (2026-04-28, CC13)

Five new entries to DECISIONS.md:

1. **License recovery via Resend + email is Beacon's chosen mechanism for lost-license-key scenarios.** Anti-enumeration POST endpoint always returns 200; the email is the actual feedback channel. License keys email back to the address the buyer used at purchase. Not in-app self-service via "show me my key" because the key + email pair is THE encryption secret for `.bcnbkp` backups.

2. **Verify-attestation lives on a dedicated subdomain `verify.clearpathedgroup.com`, not the marketing domain.** Closes the round-4 SCUTA pitch + lawyer Q28 about "vendor witnessing themselves on marketing domain." Cloudflare Pages middleware redirects subdomain root → `/verify-attestation`; CSS standalone-mode strips marketing chrome.

3. **PDF receipt UUID is client-generated; server-set `generated_at` is authoritative.** PostgREST `Prefer: return=representation` was incompatible with anon-INSERT-only RLS (INSERT-then-SELECT failed at the SELECT half). Switched to `Prefer: return=minimal` + client UUID + omit `generated_at` from client INSERT body. Server's `DEFAULT now()` sets the authoritative chain-of-custody timestamp. CHECK constraint with `±5 min from now()` blocks fabricated past timestamps from any role.

4. **Cloudflare Pages legacy `cloudflare/pages-action@v1` is deprecated and silently fails to register new functions on existing projects.** New functions added after the project's "first deploy snapshot" silently don't route. Migrate any new deploy chain to `cloudflare/wrangler-action@v3` + `wrangler pages deploy`. If migration alone doesn't unblock new functions, the project's function manifest may be stuck — recreate the project as the cleanest fix.

5. **PWA cache-bust pattern: build emits `version.json`, page polls + shows "New version available" toast.** Standard PWA upgrade pattern — replaces "users must clear browser data" footgun. Per-version dismiss key. Service worker honors `SKIP_WAITING` postMessage. Build-stamp via `vite.config.js` plugin + `define: __APP_BUILD_VERSION__`. Reusable for Apex, Waypoint, Investigator Toolkit.

## Lessons worth keeping

- **The "client UUID + server timestamp" attestation pattern is the right shape for anon-INSERT-only audit logs.** Tried `Prefer: return=representation` first; failed because anon SELECT was intentionally denied. Switching to `Prefer: return=minimal` + client-generated UUID + omitted `generated_at` (let DEFAULT now() set it) made the chain work, kept anon-INSERT-only RLS intact, AND removed the client's ability to spoof timestamps. Combined with the CHECK constraint, this is hard to beat for an "audit log without auth" pattern.
- **Cloudflare Pages legacy `pages-action@v1` is silently broken for new functions on existing projects.** Spent ~30 min diagnosing — wrangler-action@v3 is the supported successor. If wrangler also doesn't unblock, the project's manifest is stuck and the cleanest fix is delete-and-recreate (Option A). Document this so future agents don't repeat the journey.
- **Marcia's 84% with "ship and call it 1.0" is the strongest 1.0 signal we'll get.** Three rounds of patches each closed her named gaps; the trajectory shape (40 → 58 → 82 → 84) is now flattening. That's the right shape for a 1.0 release. Stop adding features for two months and harden what's there.
- **The Adversarial multi-voice audit decision (CC12) keeps validating itself.** Round 4 caught my regression (verify URL → 404 closed, but new fixes introduced their own regressions like "last sync" wording confusion). Round 4 also produced the most actionable single-finding cluster yet: 3 of 8 fixes were caught independently by 2+ voices. Run the cadence on every defensive-claims release.
- **Daubert isn't the right framing.** Lawyer round 4 self-corrected: Daubert applies to expert testimony, not documentary evidence. The actual standard for documents is Rule 803(6) trustworthiness + 901 authentication. Beacon now passes admissibility; the remaining argument is weight (jury). Any future audit framing should use Rule 803(6) language, not Daubert.
- **PWA cache-bust footgun is serious ops debt at scale.** Marcia called it out: "going to make support tickets miserable when they have 200 customers." Shipped the version.json + toast pattern this session. Should be reused by every PWA in the suite (Apex, Investigator Toolkit, Waypoint admin) — same five-minute integration each.
