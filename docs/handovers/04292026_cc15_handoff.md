## Lessons worth keeping

- **Round 2 produced six NEW convergent findings on top of the six that were closed.** The audit cadence's value is not just validating closures — each round surfaces a new tier of wounds that the prior round's patches expose. Beacon went through 4 rounds for this reason; Navigator should expect at minimum a round 3, probably a round 4 before "1.0."

- **The "snapshot toggle" pattern is a tradeoff trap.** Marsha asked for a Hearing Packet option to exclude edit-history. Reyes (round 2) had just on-record-conceded that the disclosure makes the chain admissible. Adding the toggle would have re-opened Reyes's spoliation attack — Marsha's UX preference would have cost the legal posture. Documented as DECISIONS entry: not a snapshot toggle. Worth flagging this kind of cross-voice tension whenever a single voice's polish ask is in scope.

- **Silent-overwrite is the fail-mode that produces 8-week shadow-bugs.** CC14 found it in audit_log; CC15 found it again in parent_notified_at. Same shape: a server-side authority quietly replaces a client value with no signal. Now baked as a DECISIONS principle — any column under server control raises P0001 on client write attempts. Worth a sweep across the rest of the schema to catch other silent-overwrite spots.

- **Adopting another agent's claim without verification is its own bug.** Chen's round-1 "audit.js silently rejecting every Waypoint write since 035" was carried into the CC14 handoff verbatim. CC15 verification proved the claim was misdiagnosed (no callers, not silent-rejecting). The audit cadence catches things — but the tier-A "verify before patch" step catches the audit cadence's own miscalls.
