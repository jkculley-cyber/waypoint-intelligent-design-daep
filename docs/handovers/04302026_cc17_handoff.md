### 1. Cody-driven retention fix (commit `3adc6e4` on `clearpath-apex`)

- **File whitelist + error rewrite**: added `.3gp` / `audio/3gpp`, size error leads with iPhone Voice Memos compression fix, added iCloud-placeholder hint ("if your Voice Memo is in iCloud, tap play once before uploading — that pulls the file local so the upload doesn't stall").

## Decisions logged (2026-04-30, CC17)

Five new entries in DECISIONS.md (AI Agent Team section):

1. **Adversarial multi-voice audit cadence applies to Apex too, not just Beacon/Navigator.** Confirmed value: Apex went TENUOUS/BLOCKED → DEFENDABLE/CONDITIONAL across 3 rounds, matching the Beacon and Navigator inflection points. The "uniquely Apex" voice — observed teacher (Jennifer Martinez) — is not optional; teachers are labor-protected employees with HR-evidence stake distinct from Beacon's students or Navigator's families. Reusable for any product where the documented subject has independent legal standing (Apex teachers, future suite products documenting paraprofessionals or coaches).

2. **Bias guardrails should be principle-based, NOT enumerated lists in visible system prompts.** R1 Patch 2 enumerated 24 personality descriptors as forbidden terms. R2 audit caught two convergent attacks the enumeration enabled: (a) Wallace SEVERE — discoverable in litigation as "vendor documents bias-hiding mechanism" Daubert attack; (b) Jennifer R2-N1 HIGH — prompt injection surface where a principal's notes argue clinical necessity for listed terms creating dual instructions. R3-1 fix: replace enumeration with principle ("Personality descriptors are not part of professional coaching language. Describe observable, functional teacher behavior, never personality, temperament, or affect. Recover the underlying observable behavior — specific words, specific action, specific timing.") No list to discover; no list to inject around. Reusable for any AI product where bias mitigation needs to be in-prompt — never enumerate; always principle.

3. **Constraint-tightening migrations on lifecycle-bearing columns must ship retention/cleanup infrastructure in the same batch.** Migration 015 added `ib_observations.audio_path` without retention or delete API. Chen R2 NEW HIGH: liability EXPANDED post-R1 because now Apex knows where the audio is but can't programmatically delete it. R3-4 fix shipped together: schema column (mig 015) + retention preference (mig 016 `principals.audio_retention_days`) + listing function + clearing function + bulk-delete function (RLS-gated) + cleanup Edge Function (service-role-key-gated, idempotent) + scheduling guidance. Pattern: any migration adding a column that points at external lifecycle-bearing storage (audio, video, files, attachments) requires the corresponding management primitives in the same shipping batch — never ship the path-storing column without the path-deleting infrastructure.

4. **Whisper transcription redaction defense pattern: 3-layer (Whisper-prompt + post-process regex + caller-provided exclusion list).** Whisper's `prompt` parameter is a hint, not a hard filter — over-trusting it as a redaction primitive is the trap. Post-process regex is the load-bearing layer. R3-5 added the third layer: caller (`transcribe-observation`) fetches the principal's registered teacher names from `teachers` table and passes them as exclusion list to `redactStudentNames()`. This lets the regex over-redact unknown names by design (FERPA > false positives) while preserving principal-side staff name references in their own notes. Reusable for any product where audio transcripts contain mixed adult+minor identifiers (Beacon counselor sessions if expanded to audio, Apex IB observations, future products).

5. **R3 audit inflection point isn't always "ship-it" — sometimes it's "tier upgrade only."** Beacon and Navigator hit ship-it (82-84%) at R3 because their R3 patches were retention loops (audit logs, edit history, teacher-side surfaces). Apex hit DEFENDABLE/CONDITIONAL at R3 (legal verdicts moved one tier each) but Karen still at 74% and Jennifer at 48% because R3 patches were defensive (FERPA, legal liability, retention infrastructure). Pattern: don't stop the audit cadence at "verdicts moved one tier" — only stop when the user-side score also flatlines into ship-it range. Apex needs R4 (Patch 1 + F2 + Patch 8) to hit Beacon-equivalent ship-it. Marsha's "stop at R3" rule on Navigator was specific to Navigator's user-side score also being in ship-it range, not a generic stopping rule. Recursive corollary: when the user-side score flattens before legal verdicts move (hypothetical), keep going on legal infrastructure.

## Lessons worth keeping

- **Adversarial audit cadence works on Apex too.** R1 → R2 → R3 produced the same trajectory shape as Beacon and Navigator: convergent findings → patches → tier upgrades on legal/procurement verdicts. The cost is ~75 min agent time per round (4 voices in parallel). For a product whose value prop touches teacher careers, the audit reveals architectural risks user testing won't catch.

- **Bias guardrails should be principles, not lists.** I shipped R1 Patch 2 with a 24-term enumeration, expecting it to close Jennifer's bias concern. R2 caught the new SEVERE finding I introduced — the list is now litigation evidence and a prompt-injection target. R3-1 replaced with principle. Lesson generalizes: any AI guardrail that enumerates forbidden behaviors becomes a target. Use principles, examples, and behavioral specifications instead.

- **Migration 015 was the audio_path mistake.** Adding the column without the cleanup infrastructure expanded liability per Chen R2 HIGH. The fix (mig 016 + apex-audio-cleanup function) is a lot more work than just adding the column. Pattern reminder: lifecycle-bearing columns require lifecycle infrastructure shipped together. Same trap as the constraint-tightening one from CC16 (constraints require normalization not just NULL backfill).

- **R3 inflection point is product-specific.** Beacon and Navigator hit ship-it at R3 because their R3 patches were retention loops. Apex hit DEFENDABLE/CONDITIONAL at R3 (legal verdicts up one tier) but only 74% Karen because the retention patches were deferred. Don't apply Marsha's "stop at R3" rule generically — it was specific to Navigator's user-side score also being in ship-it range. The cadence-stopping rule is "user-side score flatlines into ship-it range AND legal/procurement verdicts moved" — both, not either.

- **Cody-driven UX bug fixes feed adversarial audit framing.** The Cody retention failure was the immediate trigger that surfaced the IB voice recording bug, which surfaced the broader question "what else is silently broken or insufficient about Apex," which led to the audit. User-reported failures are the canary; adversarial audit is the systematic exploration.
