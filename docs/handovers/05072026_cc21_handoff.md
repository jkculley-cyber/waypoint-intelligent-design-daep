## Lessons worth keeping

- **Demand-pull beats build-push for unproven product directions.** A 4-week build is cheap to plan but expensive to execute against the wrong target. Nicole's conversation will tell us whether Path A or Path B is right; building Path A speculatively might burn 4 weeks against a target that isn't there. Today's $0 cost in audit-only mode is much better than the 4-week cost in speculative-build mode.
- **Marketing site is a sales lever even when product is paused.** Pulling Apex acquisition CTAs while promoting Beacon and Navigator costs zero engineering hours but should measurably shift inbound interest. The "Coming Soon" treatment also signals discipline rather than abandonment — important for existing principals who might worry the product is dying.
- **Strategic pause is harder to maintain than tactical pause.** Saving the Path A audit + discovery questions + DECISIONS.md entry + memory update prevents a future session from re-spinning the build by mistake. Without those four artifacts, the same audit would be re-spiked from scratch in 30 days when conversation context is gone.
- **Trust the codebase before reading memory.** Beacon's `db.js` adapter being already polymorphic was a happy surprise — memory said cloud mode was "aspirational" but the actual code is much further along than that. Reading code beats reading memory when trying to scope cloud-mode work.
