### 3. TOCTOU close (migration 080 + Edge Function update)

**The gap:** migration 078 + the Edge Function's `daep-documents.list({search})` check confirmed the document URL resolved to a non-zero file at approval time but captured nothing about *what file* was seen. A bad actor with Storage write access could swap the file at the same path after approval. The audit packet six months later would still point at "a file" — just not the one approved.

**The fix shipped end-to-end:**

| Layer | Change |
|---|---|
| **Migration 080** | `compliance_override_requests` gets `document_sha256` / `document_size_bytes` / `document_mime` / `document_verified_at` columns. CHECK constraint enforces hash shape (lowercase hex, 64 chars) when set. NULL allowed for pre-migration rows; new rows from the RPC are required. |
| **`fn_approve_compliance_override`** | Signature rebuilt with 4 new params (default NULL). RAISES `P0001` "Document attestation required" if hash/size/verified_at are NULL. Dual-signature check still fires first so existing smoke tests of same-user rejection don't need attestation values. |
| **Edge Function** | After document validation, downloads file via `admin.storage.from('daep-documents').download(docPath)`, computes SHA-256 via `crypto.subtle.digest('SHA-256', bytes)`, calls RPC with all 6 params, returns hash + verified_at in success body so frontend can display them. Authoritative size now uses `bytes.byteLength` not Storage metadata (defends against stale metadata). |
| **`useCompliance.js`** | Deleted unused `approveOverrideDirect` bypass hook — the exact path Reyes flagged. No callers; deletion confirmed via grep. |
| **Schema-drift CI** | `scripts/check-schema-drift.mjs` EXPECTED list extended with the 4 new attestation columns. 11/11 PASS in production. |
| **`smoke-test-073-079.mjs`** | Updated 078-3 to pass dummy attestation params so the DB-layer test still works post-mig-080. |
| **`smoke-test-edge-function-approve.mjs`** | EF-7 happy path now asserts: function-returned SHA-256 matches locally-computed `createHash('sha256').update(minimalPdf).digest('hex')` ✓, `document_sha256` + `document_size_bytes=299` + `document_mime='application/pdf'` + `document_verified_at` all persist on the row ✓. New EF-8: direct RPC call without attestation rejected with "Document attestation required" — bypass closed at DB layer, not just function layer. |

Production state: migration 080 applied, Edge Function redeployed. **Every new override approval row now carries a SHA-256 of the supporting file computed at approval time.** A hearing officer pulling the audit packet 5 years later can re-hash the file at the URL and confirm it is the same bytes the approver verified. Match = original; mismatch = swap detected.

**What was deliberately not done:** Reyes called out a follow-up — content-hash also belongs in a TOCTOU-immune location (e.g., write-once Storage backup, immutable audit_edit_log table). That's the remaining T3 item (~3-5 days) and is out of scope here; the hash on the row plus the row's audit_log trigger raises the friction floor materially without taking on that work.

## Decisions logged (DECISIONS.md)

**2026-05-11 — Architecture & Technology:**

1. **SHA-256 attestation required on every compliance override approval row. Direct RPC approval is blocked.** Migration 080 makes the Edge Function the only path to approval — `fn_approve_compliance_override` raises `Document attestation required` if hash/size/verified_at are NULL. The function downloads the supporting file at verify time, computes SHA-256 of the bytes, writes the hash + size + mime + verified_at into the override request row in a single transaction with the approval. Reyes CC22 R3 residual closed. Pattern reusable for any approval workflow with external-document evidence.

2. **Bucket-policy / function-policy MIME mirroring is correct defense-in-depth, not redundancy.** The `daep-documents` bucket's `allowed_mime_types` whitelist is identical to the Edge Function's `ACCEPTED_MIMES` set. From outside, a file that survives upload necessarily passes the function check, so the function-layer MIME check is genuinely unreachable. This is correct architecture — if the bucket policy is ever loosened (e.g., adding `image/gif` for screenshots), the function still rejects them. Documented in `smoke-test-edge-function-approve.mjs` header. Reusable pattern for any layered MIME enforcement (Beacon attestations, Apex audio, Navigator referrals).

3. **`approveOverrideDirect` bypass hooks are an anti-pattern in audit-enforced workflows. Delete them.** `useCompliance.approveOverrideDirect` was defined but never called — described in source comments as "emergency / admin recovery path." It was the exact bypass Reyes flagged. Per CLAUDE.md "if certain something is unused, delete it." Generalized rule: any function that bypasses a load-bearing chain-of-custody enforcement must either have explicit production callers or be removed; "emergency" stubs that nobody calls are net-negative because they document a hole an adversary can probe for.

4. **Staff-side and parent-side audit-history surfaces share projection logic but live in separate files. Don't pre-extract.** Parent hook reads `audit_log` via mig-079 parent RLS; staff hook reads same table via mig-035 district RLS. Projection logic (split `changes.old/new` per watched field, resolve actor names, group by entity) is 80% identical. Tempting to extract; refusing because (a) the WATCHED field lists will diverge as staff need more compliance fields than parents do, (b) the FERPA copy diverges across both rendered PDFs, (c) refactoring a load-bearing parent surface (Patricia's TRULY PROTECTIVE verdict depends on it) introduces regression risk for ~30 lines of dedup savings. **Three similar lines is better than a premature abstraction (CLAUDE.md).**

## Lessons worth keeping

- **CLI redeploy resolved the dashboard-deploy JWT-gateway intermittency on first try.** Per CC22 lesson logged in DECISIONS — CLI deploy with explicit `--no-verify-jwt` is deterministic during asymmetric-JWT-key rollouts. Confirmed by 7/7 smoke pass against the CLI-deployed function 30 seconds after deploy. Pattern reusable for next Waypoint / Apex / Beacon Edge Function deploy.
- **Bucket-policy + function-policy MIME mirroring made EF-6 untestable from outside.** Caught this on the first smoke run when text/plain upload failed at the bucket layer. Documented as defense-in-depth rather than as a smoke-test failure. Lesson: when defense-in-depth layers exactly match, the inner layer is intentionally redundant; don't try to widen the outer to test the inner.
- **TOCTOU close via in-row hash + RPC enforcement is the minimum viable chain-of-custody fix.** Three architectural pieces (hash + size + verified_at) on the row, captured in a single transaction with approval, and the RPC raises if any are missing. The remaining work (immutable audit_edit_log, write-once Storage backup) is hardening; the substrate-layer fix here is itself defensible against Reyes-style cross-examination because the hash is self-authenticating evidence. **Don't let "the perfect" delay shipping a defensible base layer.**
- **Closing-process commit step from CC15 paid off again.** CC22 audit_log shape and CC16 race column drift were each caught only after the deployed state diverged. Tonight's schema-drift CI run (11/11 PASS) confirms the 4 new attestation columns are reachable via PostgREST in production exactly as declared. Schema-drift CI now monitors 11 tables; the tripwire is doing what it was written for.
