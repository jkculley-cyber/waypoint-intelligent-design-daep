## What was done

### 1. T3 Phase 1 — audit_edit_log WORM mirror (migration 081 + hotpatch 081a)

Shipped as planned in CC23's T3 backlog. Single migration with table + lockdown + write function + bridge trigger + two verify functions + full-history backfill.

| Piece | Detail |
|---|---|
| **`audit_edit_log` table** | 15 columns including `seq BIGSERIAL`, `audit_log_id UUID UNIQUE`, full mirror of audit_log fields, `row_canonical TEXT`, `prev_row_hash TEXT` (NULL only for seq=1), `row_hash TEXT` with CHECK constraints on hash shape. Indexes on `audit_log_id`, `(entity_type, entity_id)`, and `(district_id, audit_created_at DESC)`. |
| **Lockdown** | `REVOKE INSERT, UPDATE, DELETE, TRUNCATE` from `PUBLIC`, `anon`, `authenticated`, `service_role`. Only `fn_append_audit_edit_log()` SECURITY DEFINER (owned by `postgres`) can write. RLS read: `waypoint_admin` everywhere, district `admin`/`principal` scoped to their own district. |
| **`fn_append_audit_edit_log(UUID)`** | Acquires `pg_advisory_xact_lock(hashtext('audit_edit_log_chain'))` to serialize concurrent appends. Builds canonical text via `jsonb_build_object(...)::text` (JSONB-normalized; keys sorted, whitespace canonical) with `to_char(... 'YYYY-MM-DD"T"HH24:MI:SS.US"Z"')` for microsecond-precision UTC ISO-8601 timestamps. Reads `prev_row_hash` from current chain head, computes `sha256(prev_hash || canonical)` via `encode(digest(...), 'hex')`. Idempotent — `UNIQUE(audit_log_id)` + explicit P0001 if row already mirrored. |
| **Bridge trigger** | `AFTER INSERT ON audit_log` → `fn_mirror_audit_to_edit_log()` → `fn_append_audit_edit_log(NEW.id)`. Same transaction as the originating mutation; rollback propagates. |
| **`fn_verify_audit_edit_log_chain()`** | Walks every row in seq order, recomputes expected `row_hash` from stored `prev_row_hash` + `row_canonical`, reports `(first_broken_seq, broken_count, total_count, head_hash, head_seq)`. Two failure modes detected: (a) `row_hash != recomputed` (data/hash tamper), (b) `prev_row_hash != previous v_prev` (chain link broken). Cascade behavior documented in header. |
| **`fn_verify_audit_log_matches_edit_log()`** | Cross-checks audit_log ↔ audit_edit_log. Reports `(audit_log_total, audit_edit_log_total, unmirrored_count, orphaned_count, mismatched_count, first_*_id)`. Detects post-mirror DELETE or UPDATE of audit_log by recanonicalizing the audit_log row exactly as `fn_append_*` did and comparing to stored canonical. |
| **Backfill** | DO block walks `audit_log LEFT JOIN audit_edit_log ON audit_log_id IS NULL ORDER BY (created_at, id)`. Per-row EXCEPTION block — keeps migration alive on any individually-malformed historical row. At Lone Star ISD pre-CC24 volume: 246 rows backfilled in <1s. |

**Hotpatch 081a:** The `SET search_path = public` on the digest-using functions excluded the `extensions` schema where Supabase installs pgcrypto. First smoke run caught this immediately: `function digest(text, unknown) does not exist`. Patch added `extensions` to search_path on `fn_append_audit_edit_log` and `fn_verify_audit_edit_log_chain`; re-ran backfill (idempotent via LEFT JOIN guard). Production-impact window: ~15 min between 081 apply and 081a apply, during which every audit_log INSERT would have failed and rolled back originating mutations. No external traffic in this window (user was in verify-mode), so zero customer impact.

**Mig 081 source file updated to bake the hotpatch in** — future re-applies don't need 081a.

| Piece | Detail |
|---|---|
| **`audit_chain_backup_state`** | Singleton row (`CHECK (id = 1)`), pre-seeded `(1, 0)`. ## Lessons worth keeping

- **SECURITY DEFINER + extension calls = `SET search_path = public, extensions`.** Captured in DECISIONS as a generalized rule. Audit existing migrations for the same trap as a Tier-2 task. Bare `public` is a footgun the moment any extension function is called from a SECURITY DEFINER body — and Supabase puts pgcrypto / vault / pg_net / etc. in `extensions`, never `public`.
- **Smoke tests aren't just verification — they're the production-impact-shortener for SECURITY DEFINER changes.** The 081 backfill would have aborted every mutation under traffic. We caught it in ~15 minutes via smoke. Same bug under traffic would have caused a customer-visible outage measured in hours.
- **CC18 anti-ping-pong worked.** Three rounds with the gateway, then stopped and flagged. The substrate is shipped + smoked + drift-clean. Phase 2's last 10% needs an external condition (Supabase gateway cache release) that we can't drive from inside a session.
- **Phase 1's cross-check function provided stronger evidence than CC23 closeout-discipline assumed.** It verified byte-identical canonicalization across 248 real production audit_log rows — exactly the work Phase 2's hourly verification will do automatically. That evidence is what made it safe to ship Phase 2 same-session rather than wait a week as I had originally proposed.
- **The chain alone is detection; chain + Storage backup is prevention.** Don't sell Phase 1 alone as "tamper-proof." Reyes's residual closes fully only when Phase 2 attestation is actually running. The substrate is shipped, the cron is wired, and the gateway block is a Supabase-side intermittency that has historically resolved itself within a session window or two.
