## Lessons worth keeping

- **CI tripwires for row invariants cost ~10 lines and prevent multi-round prod debugging.** The schema-drift CI extension to `REQUIRED_ROWS` was ~30 lines including the error-message scaffolding. CC26 spent multiple rounds debugging because the audit sentinel row had drifted out of production silently. The cost asymmetry is enormous: a missing-row CI fail is a 10-second discovery; a missing-row production discovery is a multi-round debugging cycle with a real seed-rerun in the middle. Pattern: any "must exist for the app to work" row is a candidate (sentinel UUIDs, baseline configuration rows, required FK targets the app reads but never writes). Don't preemptively add all of them — add each one after its first failure mode is concretely understood.

- **Hero carousel dedupe was a 3-line change × 4 slides — the analysis was the work.** The actual edits were trivial (replace `<div with 2 buttons>` with `<div with 1 button>`). The thinking time was: which CTA wins per slide, which destination is highest-intent given the user's funnel goals ("get leads for waypoint"), what stays as the cards-below pattern. When you're 90% of the way through a marketing-site cleanup, the writing of the new CTA copy is the load-bearing decision, not the HTML mechanics. Worth noting because the same pattern will hit the still-deferred Beacon district CTA + Apex waitlist form (when those unpause).

- **Lead capture trumps one-click-to-creds for B2B SaaS demos.** CC25 logged the rule "demo CTAs must land on creds within one click." This session's Waypoint lead-gate form refines that: the form is one click *to the form*, then one form submission *to the creds* (with creds revealed inline + auto-emailed). Two clicks. The trade is: we lose ~10% of impatient visitors who bounce on the form, but every visitor who submits is a qualified-by-form-completion lead in the CRM with role, district size, and contact info. For Waypoint where Marlene-class buyers need 30-day-cycle nurture sequences, that's the right trade. Generalizable: when the buyer cycle is long (district sales, enterprise eval) prefer lead-gate; when the buyer cycle is short (individual purchase, free-trial-only) prefer one-click. Navigator + Waypoint = lead-gate; Beacon + Investigator Toolkit = one-click (already in place via Cloudflare-protected download gates).

- **Three identical-shape lead-gen forms is the cue to extract.** `navigator.html` had the first form (CC25), `waypoint.html` is now the second (CC27). When the third Clear Path product needs a lead-gen form (next will probably be Apex when it unpauses), extract the shared form fragment + 3-fetch JS into a partial — at N=3 the duplication is real and the shape has stabilized enough to design the abstraction. At N=2 the duplication is ~80% but the per-product variation (color scheme, district-size buckets, role list, success-card colors) is enough to make extraction premature.

- **`HTTP 200` from an email-sending Edge Function only proves the API queued — not that gmail delivered.** Resend message ID returned means Resend's API accepted; Resend may then still bounce, classify-as-spam, or have the recipient's gmail tab-route the message into Promotions. User-side, this looked like "I only got 1 email" but was actually "tab routing on the welcome email." Always tell users to check Promotions/Updates tabs + spam folder + Resend dashboard when an expected email doesn't appear in Primary. Resend dashboard exposes Delivered / Bounced / Complained / Queued / Failed status by message ID, which is the authoritative answer.

- **Spreadsheet round-trip corruption is a known LibreOffice quirk, not a user error.** The DAEP v17 audit found `MAXIFS` → `_xludf.maxifs` + `LET` → `_xludf.let` + uppercase `ARRAYFORMULA` → lowercase `arrayformula` — these are all symptoms of a LibreOffice/Calc-or-Google-Sheets round-trip on a .xlsx that originated in real Excel. Any time a user reports "this worked, then I sent it through a different spreadsheet program, now it's broken in Excel," check for these three corruption patterns first. Fixable by stripping prefixes and replacing `arrayformula()` with per-row Excel formulas. Total fix time for the user's v17 → v18 was ~90 minutes; figuring out the pattern saved hours over a brute-force formula-by-formula rewrite. Memory entry `feedback_audit_v4_seed_first.md` already covers the related "audit seed formulas first" lesson — this LibreOffice round-trip lesson is a distinct adjacent gotcha worth its own pattern recognition.
