## Lessons worth keeping

- **Feature with multiple entry points → grep every one.** CC19 added date/time + edit-history to `CommunicationsPage`. The per-student `LogContactModal` on `StudentDetailPage` was missed — same `db.insert('communications', ...)` shape, different file. Nicole found it months later. Pattern: when shipping a UX improvement to a feature that has multiple entry points (different pages, modals, embed locations), grep the underlying write call (`db.insert('communications'` here) — that immediately surfaces every render site of the same shape. The "shared field shape, separate UI" pattern is the trap.

- **`<select>` text color doesn't extend cleanly to `<option>`.** Three marketing-site role dropdowns inherited `--text:#f0e6d3` from the `<select>` and rendered cream-on-white in the native OS panel — invisible until hover painted them blue. The bug is invisible in design tools because option styling only renders in the OS-native dropdown. When building forms on dark backgrounds, always explicitly style `select option {background:...; color:...}`. `index.html` had this handled per-option, `navigator.html` / `waypoint.html` / `download-gate.js` did not — and the `download-gate` modal was the worst offender because the gate is the *only* thing a download visitor sees.

- **The closing process is load-bearing.** CC27 shipped real work (3 production-affecting changes + 14/14 schema-drift CI improvement) but no session-context update, no handoff, no ops sync. Today's opening process caught it because the git log showed a CC27 commit but no matching handoff file. The cost was small (~5 min to backfill from the commit message) only because the commit message was thorough. Lesson: even short sessions need the closing process, or skip it deliberately and acknowledge it in the next session's open. Silent skips create stale docs that future sessions waste time reconciling.

- **First district signal for Beacon → resist build temptation.** User reported "district intrigued with Beacon." Natural response is "great, let me start building cloud mode." The demand-pull rule from 2026-05-07 exists precisely to break that reflex. The right move was 30 seconds pointing at the discovery questions doc and stopping. Saved a 4-week speculative build that might have targeted the wrong shape.
