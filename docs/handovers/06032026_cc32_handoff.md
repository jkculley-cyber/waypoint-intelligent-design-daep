## Lessons worth keeping
- **Audit the shared substrate, not just the product's own files.** Third time a "missing" finding was actually present elsewhere: CC31 Beacon endpoint (sibling repo), CC30 Navigator chain (shared `audit_log`), now confirmed. When a finding says "X doesn't exist," confirm the search covered the shared/sibling surface **and verify against live state** before building. A 3-day "port" was one read-only query away from being recognized as already-done.
- **Verify before building; the SQL Editor is enough when the key is revoked.** The whole reshaping turned on read-only diagnostics run in the SQL Editor (single-query consolidation, since the editor returns only the last statement). No code was written against an unverified assumption.
- **Don't fabricate data into an append-only chain to make a demo look clean.** Honest per-record reporting beats a permanent one-way-door write.

## Addendum 2 — Beacon B-2 RESOLVED (same session)

Took on B-2 (the Beacon verify-attestation endpoint) and resolved it end-to-end on prod.

**Diagnosis:** the endpoint code (`clearpath-site/functions/api/verify-attestation.js`), the verify page, and Beacon's printed URL (`pdfAttestation.js:102` → `clearpathedgroup.com/verify-attestation?hash=`) were all correct. The write path was verified live (anon INSERT to ops `pdf_attestations` → 201; anon read RLS-blocked). The only break was Cloudflare Functions routing.

**Real root cause (deeper than the dashboard-config theory):** the GitHub Action `deploy-clearpath-site.yml` ran `wrangler pages deploy clearpath-site` **from the repo root**, so wrangler looked for `./functions` at the root (nonexistent) and uploaded the marketing site **without** the `/api/*` Functions. That direct-upload deploy fires on every `clearpath-site/**` push and raced/overwrote the dashboard git-integration build → the endpoint flapped between working and 404.

**Fix (3 parts):**
1. **Durable, in-repo (`19496be`):** `workingDirectory: clearpath-site` + `command: pages deploy .` in the Action → wrangler now bundles `clearpath-site/functions/`. Every push routes Functions.
2. **Infra (user, dashboard):** cpeg-site Root directory → `clearpath-site` (git-integration path) + `OPS_SUPABASE_SERVICE_ROLE_KEY` Production secret on cpeg-site = the **Supabase `xbpuqaqpcbixxodblaes` `service_role` JWT** (legacy keys still active there).
3. Removed the temporary `?debug=1` instrumentation I'd pushed to unmask the masked-502 (`68b9dae` added → `ff1beef` removed).

**Process:** validated Config B on a **throwaway Pages project** first (zero risk) — all 4 routing checks passed — before promoting. Pre-registered a known-good test hash (`c`×64) for instant verification.

**Verified live on prod (stable across many checks):** `/api/ping`→JSON · registered hash→`found:true`+row · novel hash→404 `found:false` · verify page→200 · homepage intact · `?debug=1` inert.

**Commits to `main`:** `68b9dae` (diag) → `ff1beef` (cleanup) → `19496be` (workflow fix) → `28b7e4b` (playbook + risk register).
