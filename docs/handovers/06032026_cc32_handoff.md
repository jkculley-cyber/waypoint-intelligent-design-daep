## Lessons worth keeping
- **Audit the shared substrate, not just the product's own files.** Third time a "missing" finding was actually present elsewhere: CC31 Beacon endpoint (sibling repo), CC30 Navigator chain (shared `audit_log`), now confirmed. When a finding says "X doesn't exist," confirm the search covered the shared/sibling surface **and verify against live state** before building. A 3-day "port" was one read-only query away from being recognized as already-done.
- **Verify before building; the SQL Editor is enough when the key is revoked.** The whole reshaping turned on read-only diagnostics run in the SQL Editor (single-query consolidation, since the editor returns only the last statement). No code was written against an unverified assumption.
- **Don't fabricate data into an append-only chain to make a demo look clean.** Honest per-record reporting beats a permanent one-way-door write.
