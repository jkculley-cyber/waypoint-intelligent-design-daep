## 2. Beacon-parity 14-day trial purchase-gate

User: *"people will need to purchase license after 14 day trial like beacon."* Chose **full Beacon parity** when asked about scope.

The toolkit had three holes:
1. **Re-trial loophole** — the expiry screen still rendered "Start Free 14-Day Trial"; clicking it wrote a fresh `inv_trial` → infinite re-trial, never purchase.
2. **Trial users couldn't create cases** — `intake.js` gated on `checkLicense().softGated`, which is `true` for **any** user without a license key (including active-trial users).
3. **Hard lockout at expiry** — the app refused to boot without a license, trapping the user's own on-device case data.

### What shipped (matches Beacon's data-preserving model)
- **`license.js`** — single source of truth for trial state (`getTrialInfo`, `startTrial`, `hasUsedTrial`, `isTrialActive`, `getTrialDaysRemaining`) + a **synchronous `isGated()`** master gate: *active paid license OR active trial ⇒ not gated*. `startTrial()` is **one-time** — never resets an existing/expired `inv_trial` (the on-device analogue of Beacon's `trial_started_at`).
- **`db.js`** — `put()`/`del()` throw `"Your free trial has ended. Enter a license key in Settings to continue."` when gated, **except** the `settings` + `audit_log` stores, so a gated user can still **view, export, import, and finish setup** — only new/edited case records are blocked. (Export/import use raw store ops, not the gated helpers, so they keep working.)
- **`intake.js`** — gates on the trial-aware `isGated()` (fixes hole #2).
- **`main.js`** — trial helpers moved to `license.js`; the trial button renders only when the trial was never used; at expiry the app loads **gated** (soft-gate banner + Subscribe) instead of hard-locking; convert in-app via **Settings → Activate**.

Shipped **[investigator-toolkit#1](https://github.com/jkculley-cyber/investigator-toolkit/pull/1) (`5497e71`)** (source → hosted app auto-deploys) + **[waypoint#20](https://github.com/jkculley-cyber/waypoint-intelligent-design-daep/pull/20) (`3ae255e`)** (rebuilt offline download).

### Verification (real browser)
Ran the toolkit's own dev server via a new **untracked** `.claude/launch.json` `toolkit-dev` entry (`npm --prefix ../investigator-toolkit run dev` on port 5182) and drove it:
- fresh load → trial entry + trial button ✅
- start trial → app loads, no gate, "14 days remaining" ✅
- **expired** (set `inv_trial.started` 20 days back) → app **loads** (no lockout), soft-gate banner + Subscribe, `put('cases',…)` **throws** the gate message, `put('settings',…)` + `exportAllData()` still work ✅
- simulated active license → `isGated()` false, writes succeed ✅
- re-trial button unreachable after first use ✅

Then confirmed the gate string (`"Your free trial has ended"`) live in **both** the hosted app bundle and the deployed download.
